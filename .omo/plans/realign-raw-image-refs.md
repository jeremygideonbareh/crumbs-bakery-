- **Wave 1 — Image identification + asset prep:** worker inspects 4 WhatsApp files (the 3 July-18 ones via fresh-bakes-2/3/4.jpeg derived copies OR the raw originals via file:// browser render fallback — look_at is currently down; worker tries `look_at` first, triggers fallback if it errors); records identification verdicts at `.omo/artifacts/team-photo-identification.md`. This todo can run as a single worker; the inspection and the script update are sequential within that worker.
- **Wave 2 — Code swaps (parallel, three independent files):** three workers run concurrently — one for each file. The products.js worker handles 3 changes in one pass (syntax fix + 4 Pexels swaps + 1 dedup edit); the contentDefaults.js worker handles 1 Pexels swap; the AboutPage.jsx worker consumes the identification verdict from Wave 1 to apply the team-array fix.
- **Wave 3 — Verification:** build, grep audit, file-existence audit, 5 Playwright screenshots, F1-F4 verification wave. Receipts in `.omo/artifacts/final-verification.md`.

Every Conventional Commit uses prefix `feat(image-realign)`, `chore(image-realign)`, or `docs(image-realign)` per the action. Atomic commits per todo. Worker creates `feat/image-realign-v2` branch from current `main` to keep this plan's commits distinct from the prior `feat/image-swap` commits.
